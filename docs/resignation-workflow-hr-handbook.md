# 離職流程操作手冊(人資版)

對應系統:CHUWA HR 系統
適用對象:人資 / 老闆兼任人資 / 新進人資接手
更新日期:2026-05-26

本手冊教你從收到員工離職申請、到完成最後薪資結算的完整流程操作。

---

# 1. 員工送出申請後、你會收到什麼

當員工在系統送出離職申請:

1. 直屬主管會先收到通知 → 審核同意 / 退回
2. 主管同意後 → 執行長收到通知 → 審核同意 / 退回
3. 執行長同意後 → 人資(你)收到通知 → 審核完成

你會在 dashboard「待我審批」看到提示、也會收到推播。

## 1.1 進入審核頁面

1. 從左側選單點「審批」
2. 點「待我審批」分頁
3. 找到該筆「離職申請」、點開詳情

## 1.2 審核前確認事項

點開申請、確認:

- 員工資料正確(姓名 / 部門 / 入職日)
- 預計離職日合理(員工年資 vs 法定預告期)
- 離職原因清楚
- 交接事項說明清楚
- 直屬主管 + 執行長都已通過

## 1.3 通過 / 退回

- 同意 → 點「✓ 通過」(可選擇填備註)
- 退回 → 點「✗ 退回」(必須填退回原因)

---

# 2. 通過後系統自動幫你做的事

當你按下「✓ 通過」、系統會在幾秒內自動:

1. 將員工狀態改為「已離職」
2. 設定離職生效日
3. 停止員工的系統登入權限(離職日後)
4. 自動建立 **離職檢核表**(46 項待辦清單、6 項系統會自動完成、剩 40 項由你勾選)
5. 自動結算員工特休餘額(折算成薪資)
6. 自動結算員工補休餘額(折算成加班費)
7. 自動建立離職月薪資草稿(按實際在職天數計算)
8. 推播通知你:「✅ 員工離職核准、請啟動離職檢核表」

---

# 3. 進離職檢核表

## 3.1 從哪裡進

兩種方式:

**方式 A**:你收到的推播通知、點進去
**方式 B**:到「員工列表」→ 找該員工 row → 點「📋 離職檢核」按鈕

## 3.2 你會看到什麼

頁面分 3 區:

1. **員工資料卡**:姓名、部門、入職日、離職日、原因
2. **進度條**:目前完成 X / 46 項
3. **8 大類待辦清單**:可展開 / 收合、共 46 項

## 3.3 8 大類清單

1. **HR 行政**(9 項):勞健保退保、勞退停止、扣繳憑單等
2. **薪資結算**(6 項):最後月薪、特休 / 補休折現、獎金結算
3. **系統權限撤銷**(9 項):HR 系統、公司 email、各 SaaS 帳號
4. **排班 / 出勤 / 假勤**(多項):清除未來排班、結清出勤
5. **組織關係**(多項):通知客戶、案件交接
6. **實體資產回收**(多項):電腦、識別證、鑰匙、制服
7. **工作交接**(多項):交接給誰、文件歸檔
8. **通知 Audit**(多項):法定文件交付、出口面談

每項可標:

- ⚪ 待辦
- ✓ 已完成(自動帶日期 + 你的名字)
- ⊘ 不適用(例:員工本來就沒配公司筆電)

可在每項加備註。

## 3.4 優先處理 → 法定期限

法律有明文規定的、優先處理(從離職日當天算起):

| 項目 | 期限 |
|---|---|
| 健保退保 | 3 天內 |
| 勞保退保 | 5 天內 |
| 勞退提繳停止 | 5 天內 |
| 就業保險退保 | 5 天內 |
| 服務證明書 | 員工要求時提供 |
| 所得稅扣繳憑單 | 次年 1/31 前 |

---

# 4. 結算最後月薪(最重要)

## 4.1 進入薪資頁

1. 從左側選單點「薪資」
2. 切到員工離職的那個月份(例:員工 5/13 離職 → 切到 2026/05)

## 4.2 找到該員工

找到員工 row、你會看到:

- 員工欄下方有紅色 **「✦ 離職月」** 標籤
- 月薪欄位下方有紅色小字 → `$—`(表示薪資還沒重新計算)

## 4.3 點開該 row「編輯」、看 Pro-rata 結算

跳出詳情 modal、會看到:

- **✦ 離職月 Pro-rata 結算** 紅框區塊
- 在職曆日:13 / 31 日
- 原月薪:$30,000
- **Pro-rata 月薪:$—**(還沒算)
- 結算日薪:$—
- 特休未休折現:$0 或實際金額
- 補休餘額折現:$0 或實際金額

## 4.4 觸發「重新計算」

關閉 modal、回薪資列表、點「批次重算」按鈕(觸發系統計算當月所有員工)。

等系統跑完、會跳通知「處理 X 員工、成功 X」。

## 4.5 確認結果

重新點該員工「編輯」、看:

- **Pro-rata 月薪**:已算出(例:$12,581)
- **結算日薪**:已算出(例:$1,000)
- **特休未休折現**:對應金額
- **補休餘額折現**:對應金額
- 各扣繳項目(勞保 / 健保)也已按比例縮減

## 4.6 推進薪資狀態

確認金額合理後、推進薪資狀態:

1. 草稿 → 待覆核:標記為待覆核
2. 待覆核 → 已確認:覆核完成
3. 已確認 → 已撥款:薪資已撥款

每一步操作系統會記錄時間 + 操作人。

# 5. 柯郁含案例(具體範例)

實際範例供參考:

**員工**:柯郁含 / 倉儲後勤部
**離職日**:2026-05-13
**原月薪**:$30,000

系統自動處理後、5 月最後薪資:

| 項目 | 金額 | 說明 |
|---|---|---|
| Pro-rata 月薪 | $12,581 | $30,000 × 13/31(在職 13 天) |
| 全勤獎金 | $818 | 按比例算 |
| 補休折現 | $1,340 | 8 小時 × 加班費率 1.34 |
| 特休折現 | $0 | 特休 3 天已用完 |
| **薪資總額** | **$14,738** | 應發薪資 |
| - 勞保扣繳 | -$251 | 按實際投保 13 天計 |
| - 健保扣繳 | -$210 | 按實際投保 13 天計 |
| **實領(淨額)** | **$14,277** | 撥款金額 |

整套自動算、你不需要手動計算、只要按「批次重算」+ 確認金額。

# 6. 老闆兼任人資的特殊情況

如果你的公司沒有獨立人資角色、是老闆(執行長)兼任:

## 6.1 系統會自動讓你看得到

當員工離職申請走到「人資」這一站、雖然你的角色是「執行長」、但系統會自動讓你在「待我審批」也看得到、可以代為簽核。

## 6.2 簽核記錄會自動標明

當你以執行長身份代簽人資這一站、系統會在備註自動加:

> 「[執行長代簽人資步驟(公司無獨立人資)]」

這個記錄是給未來勞動檢查時可以解釋用、不影響流程。

## 6.3 之後流程一樣

跑完三站簽核後、系統一樣自動處理離職後續、進離職檢核表、薪資結算等。

---

# 7. 員工自己提錯的話怎麼辦

## 7.1 員工想取消

- 直屬主管還沒通過 → 員工自己可以在「我的申請」取消
- 直屬主管已通過 → 員工聯繫你 → 你決定是否退回(按「✗ 退回」、填退回原因)

## 7.2 員工填錯日期 / 原因

- 還在審核中 → 員工自行取消、重送一份
- 已通過 → 由你(人資)在系統內手動調整、或重新核准一次新申請

---

# 8. 緊急情況處理

## 8.1 系統沒自動建離職檢核表

很罕見、但若發生:

- 進員工列表、找該員工、應該還是看得到「📋 離職檢核」按鈕
- 點進去若顯示「該員工尚未建立離職檢核表」→ 聯繫技術支援

## 8.2 系統沒推播給你

每次有員工申請、系統會推播通知你。若沒收到:

- 確認你的瀏覽器有開啟通知權限
- 或直接到 dashboard 看「待我審批」widget 的計數

## 8.3 薪資算出來不合理

- 確認該員工 row 有「✦ 離職月」標籤
- 確認 Pro-rata section 有正確顯示在職天數
- 如有疑問、聯繫技術支援

---

# 9. 法定義務 quick reference

從員工離職日(D)起算:

| 時間點 | 該做的事 |
|---|---|
| D+0(離職當天) | 員工最後上班日、停止排班、停止打卡 |
| D+3 | 健保退保 |
| D+5 | 勞保退保、勞退停止、就業保險退保 |
| 月底 | 最後月薪結算、撥款 |
| 次年 1/31 | 所得稅扣繳憑單發放 |

---

# 10. 聯繫窗口

如系統操作有問題、聯繫:

- **內部技術窗口**(系統開發 / 維護)
- **勞動法規諮詢**(勞動部 1955 專線)
